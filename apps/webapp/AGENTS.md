# @hazard-pay/webapp

The player-facing web client: TanStack Start in SPA mode (prerendered
shell, client-side routing — Start is deliberately used lightly), rendering
`@hazard-pay/ui` components. Two realtime tiers (ADR 0004): the overworld
polls via TanStack Query stale-while-revalidate; the match tier is a
one-way SSE stream (`useTickStream` is the hello-world client half).

## Run

```
pnpm --filter @hazard-pay/webapp dev
```

Vite serves on http://localhost:5173. `pnpm dev` at the repo root runs this
alongside the Ladle workbench and apps/api (port 3000).

## Api wiring (dev): proxy, not CORS (#50, decided in #20)

All api traffic is same-origin through the Vite dev proxy
(`vite.config.ts`): `/api`, `/health`, `/overworld`, `/player`,
`/telemetry`, `/ticks` forward to `http://localhost:3000`. EventSource,
better-auth cookies, and the telemetry flush therefore need zero
client-side configuration — no configurable base URL exists in client code
(`src/lib/api.ts` uses the page origin at runtime; its localhost constant
is only a prerender-safe placeholder that never serves a request). A new
api route group means a new proxy prefix. The api must be running for
live data; every surface renders honestly without it.

## Layout

- `src/routes/` — file-based routes; `__root.tsx` owns the document shell,
  head, and the design-system body classes
- `src/routes/index.tsx` — overworld hello screen: tick counter/countdown
  poll the contract route, the Uplink panel tails the live tick stream;
  credits/heat/crew/missions are still a canned snapshot (labeled as such)
- `src/routes/match-proto.tsx` + `src/match-proto/` — #27 prototype: PixiJS
  v8 mounted imperatively (no @pixi/react, per the #26 research). The
  `MatchStageHandle` mount/teardown in `match-proto/stage.ts` is the
  reference pattern for the future real match view; sprite data and
  animation math live in pure TS modules tested in Node
- `src/lib/api.ts` — the typed oRPC client over `@hazard-pay/api/contract`
- `src/lib/use-tick-stream.ts` — THE client half of the transport seam
  (ADR 0004 §2): EventSource + `Last-Event-ID` resume, parses the envelope,
  emits `tick.received`, invalidates the `["overworld"]` query tier
- `src/lib/telemetry.ts` — browser telemetry init (ADR 0005 §6), dev-only,
  called once from `__root.tsx`
- `src/router.tsx` — MUST export `getRouter` (the Start plugin resolves it
  by name); also owns the shared QueryClient defaults
- `src/routeTree.gen.ts` — generated by the Start vite plugin on dev/build;
  committed, eslint-ignored, never hand-edited. It also emits the
  `Register` module augmentation — do not add your own.
- `src/styles/globals.css` — imports `@hazard-pay/ui/globals.css` (which
  brings Tailwind itself) + the consumer-side `@source "../";`

## Design-system contract (packages/ui/AGENTS.md has the full version)

- Never `@import "tailwindcss"` here — it arrives via the ui package CSS.
- The `@source "../";` in `src/styles/globals.css` is REQUIRED for
  Tailwind to scan this app's own sources.
- Fonts: `import "@hazard-pay/ui/fonts"` once, in `__root.tsx`.
- Semantic utilities only (`bg-panel`, `text-accent`, `font-display`…);
  no raw colors.

## Tests

- `pnpm --filter @hazard-pay/webapp test` — Vitest, node environment by
  default; DOM-dependent suites opt in per file with a
  `@vitest-environment jsdom` pragma.
- Canvas/WebGL never runs in tests: pixi.js is mocked and the tests assert
  our lifecycle contract (mount/unmount/remount, StrictMode init race).
  Keep renderable state in pure modules so it tests without a renderer.

## Query conventions

- Overworld surfaces poll; only the match-tier stream subscribes. Defaults
  live in `makeQueryClient` (staleTime 15s, refetch on focus). Per-surface
  `refetchInterval` is set at the query site, not globally.
- Query keys: `["overworld", <surface>]` — keep the first segment stable
  so a tick boundary can invalidate the whole tier at once. This is live
  policy: `useTickStream` invalidates `["overworld"]` on every tick.

## Gotchas

- SPA mode still evaluates route modules during prerender of the shell —
  keep module top levels browser-safe.
- `vite dev`/`vite build` regenerate `src/routeTree.gen.ts`; if routes
  changed, commit the regenerated file in the same commit.
